==================================================
ওভারভিউ
==================================================

এস্কর্টের ডিউটি শেষ হলে → রিলিজ স্লিপ প্রক্রিয়া শুরু হয়।
রিলিজ স্লিপ = পেমেন্টের ট্রিগার।

==================================================
ধাপ ১: ডিউটি স্লিপ → রিলিজ স্লিপ
==================================================

ডিউটি স্লিপ (Duty Slip):
  জাহাজে উঠার সময় ইস্যু করা হয়।
  তথ্য: জাহাজের নাম, মাস্টারের নম্বর, শুরুর তারিখ।

রিলিজ স্লিপ (Release Slip):
  ডিউটি শেষে সুপারভাইজার তারিখ ও সময় লিখে সই করেন।
  স্বাক্ষরিত ডিউটি স্লিপই রিলিজ স্লিপ হয়।

==================================================
ধাপ ২: WhatsApp-এ জমা
==================================================

এস্কর্ট রিলিজ স্লিপের ছবি WhatsApp-এ পাঠাবেন।
সাথে জানাতে হবে: জাহাজের মাস্টার কি খাবার টাকা দিয়েছেন?

==================================================
ধাপ ৩: সিস্টেম পেমেন্ট হিসাব (DRAFT)
==================================================

OCR নিষ্কাশন করে: জাহাজের নাম, গন্তব্য, শুরুর তারিখ, রিলিজ তারিখ, এস্কর্টের নাম।
সিস্টেম হিসাব করে:

  duty_days     = রিলিজ তারিখ − শুরুর তারিখ
  shift_payment = মোট শিফট × ৳২০০
  conveyance    = গন্তব্য অনুযায়ী (দেখুন transport_allowances.txt)
  food_advance  = duty_days × ৳১৫০ (যদি মাস্টার খাবার না দিয়ে থাকেন)
  net_payment   = shift_payment + conveyance + food_advance − পূর্ববর্তী অগ্রীম

সমস্ত হিসাব DRAFT — Admin অনুমোদনের আগে কোনো পেমেন্ট হয় না।

==================================================
মানক ক্ষেত্রের নাম (Standard Field Names)
==================================================

  duty_start_date   = প্রথম দিন এস্কর্ট জাহাজে উঠেছেন
  duty_end_date     = রিলিজ স্লিপে সাক্ষরিত তারিখ
  release_location  = মুক্তির স্থান / বন্দর (transport rate নির্ধারণে ব্যবহৃত)
  duty_days         = মোট হিসাবকৃত দিন
  vessel_name       = মাদার ভেসেলের নাম
  lighter_vessel    = লাইটার / বার্জের নাম
  escort_name       = গার্ডের পুরো নাম (স্লিপ অনুযায়ী)

==================================================
ধাপ ৪: Admin অনুমোদন
==================================================

Admin APPROVE কমান্ড দিলে → পেমেন্ট চূড়ান্ত।
Admin REJECT কমান্ড দিলে → কারণ কর্মীকে জানানো হয়।
Admin EDIT করতে পারেন হিসাবে কোনো ভুল থাকলে।

==================================================
গুরুত্বপূর্ণ: সব রিলিজ পেমেন্ট DRAFT-ONLY
==================================================

কারণ: transport rate ও food amount Admin দ্বারা যাচাই আবশ্যক।
স্বয়ংক্রিয় পেমেন্ট নেই।
