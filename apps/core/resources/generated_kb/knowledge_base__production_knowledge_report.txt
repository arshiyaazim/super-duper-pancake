## C-03 · `modules/message_router/__init__.py` — 15-Step Routing Pipeline

**Purpose:** Every inbound message passes through exactly this pipeline in order.

**15 Steps:**
1. **Step 0 (silent-skip):** If sender display name contains any skip token → log only, no reply, no draft
2. **Step 1 (dedup):** Check message seen within 120s window → drop if duplicate
3. **Step 2 (source gate):** Is source in auto_reply_sources? If not → save message only, exit
4. **Step 3 (identity):** Resolve sender role via Identity Brain (11-role priority system)
5. **Step 4 (RBAC):** Admin command check → route to admin_commands if detected
6. **Step 5 (escort order):** `_looks_like_escort_order()` → route to escort module if match
7. **Step 6 (operational block):** Operational roles (admin/accountant/employee/supervisor/family/client) → do NOT enter recruitment funnel
8. **Step 7 (recruitment check):** Is active recruitment session? Continue funnel
9. **Step 8 (intent classify):** Classify intent via rule → role-context → Ollama
10. **Step 9 (attendance):** Attendance keywords → log to DB
11. **Step 10 (payment ingest):** Looks like payment SMS → ingest pipeline
12. **Step 11 (safe auto-send gate):** Check if intent is in safe list for auto-send
13. **Step 12 (office location fast path):** Office location → KB only + hardcoded fallback, no AI
14. **Step 13 (KB lookup):** Search `fazle_knowledge_base` DB table and resources/*.txt
15. **Step 14 (reviewed reply memory):** Check if admin has previously reviewed+edited a similar reply
16. **Step 15 (AI fallback):** Generate reply via AI provider chain

**Safe auto-send intents:** `recruitment, join, greeting, office_location, salary_query, payment_due, attendance, leave, escort_duty`
**Explicitly NOT safe for auto-send:** `advance_request`
**Silent-skip name tokens:** `al-aqsa, escort, client, operation, tcis, gms, dalal, office`

## C-05 · `modules/payment_workflow/__init__.py` — Escort Payment Draft

**Purpose:** Generate payment drafts for escort programs at release.

**Formulas (ALL EXACT):**
```
daily_rate = basic_salary / 30         (fallback: ৳1,200/day)
gross = duty_days × daily_rate
deductions = food_bill + conveyance + advances (THIS program, CURRENT payroll month)
net_payable = max(gross - deductions, 0)
```

**Draft text fields:** employee name, vessel name (lighter), master mobile, program date, shift (D/N), gross amount, net amount, itemized deductions, bKash/Nagad number

**Advisory lock:** `SELECT pg_advisory_xact_lock($1)` on escort_program_id — prevents duplicate payment drafts when multiple requests arrive simultaneously

**Default daily rate:** ৳1,200/day (used when employee has no basic_salary set)

## C-13 · `modules/payment_ingest/__init__.py` — Payment SMS Ingest Pipeline

**Purpose:** Parse incoming payment SMS/notifications and stage them for matching.

**Pipeline:**
1. `looks_like_payment_sms()` OR `is_admin_cash_shorthand()` detection
2. `parse_payment_sms()` — extract method, TrxID, amount, mobile, date, sender name
3. Match employee by phone or name (using rapidfuzz)
4. INSERT into `wbom_staging_payments` (idempotency key = sha256 of method|trxid|amount|mobile|date)
5. `auto_finalize()` — promote to `wbom_cash_transactions` if conditions met

**Auto-approve conditions:**
- `name_ratio >= 0.92` (rapidfuzz match)
- `amount > 100` AND `amount <= 50,000`

**Supported payment sources:** bKash SMS, Nagad SMS, Rocket SMS, generic admin paste, cash shorthand format

# J. PAYMENT/CASH LEDGER KNOWLEDGE MAP

## J-01 · Payment Flow Types

1. **bKash/Nagad/Rocket SMS** — auto-parsed from SMS forwarded to WhatsApp
2. **Admin cash shorthand** — `Cash <phone> <name> <amount>` typed by authorized admin
3. **FPE income shorthand** — `Income <phone> <name> <amount>` for income recording
4. **Manual PAID command** — `PAID <phone> <amount> <method>` from admin

## J-02 · Payment Ingest Rules

- Auto-approve: name fuzzy match ≥ 0.92 AND ৳100 < amount ≤ ৳50,000
- Below ৳100: rejected (too small)
- Above ৳50,000: manual review required
- Name ratio < 0.92: manual review required
- Idempotency: duplicate TrxID+amount+mobile+date combination → silently skipped

## J-03 · Payment Completion Process

Per Management Decision C-09:
1. Admin sends payment info to accountant
2. Accountant confirms receipt
3. Receipt confirmation = payment complete
4. Ledger (`wbom_cash_transactions`) updates immediately

## J-04 · Advance Rules

- Guideline: ৳500–৳1,000 per request (NOT a hard cap — per Management Decision C-08)
- Recorded via `ADVANCE <phone> <amount>` command
- Deducted from net payable in payroll calculation
- Deductions: only advances for THIS program in CURRENT payroll month

## J-05 · Payroll Formula

```
daily_rate = basic_salary / 30  (fallback: ৳1,200/day)
gross = duty_days × daily_rate
deductions = food_bill + conveyance + (advances: this program, current month)
net_payable = max(gross - deductions, 0)
```

# N. RAG AND VISIBILITY GAP REPORT

## N-01 · CRITICAL: knowledge_base/ Markdown NOT Indexed

**Problem:** The RAG engine (`modules/rag/__init__.py`) indexes only:
1. `resources/*.txt` files
