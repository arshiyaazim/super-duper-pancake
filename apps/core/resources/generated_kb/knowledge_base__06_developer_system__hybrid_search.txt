# Hybrid Search System

**Article Type:** Developer System Reference
**Visibility:** Developer / Admin (Internal)
**Source Module:** `modules/rag/__init__.py`
**Production Status:** ACTIVE — enabled via HYBRID_SEARCH_ENABLED=true in .env
**Certified:** 2026-06-23 (CONFLICT-2 resolution — previous 5-signal ranking description was a design draft, never implemented)

---

## Overview

Fazle uses Hybrid RAG: BM25 keyword search combined with Qdrant vector search, fused via Reciprocal Rank Fusion (RRF). This replaced the single BM25 mode when Hybrid RAG was deployed in Session 2.

---

## Environment Control

```
HYBRID_SEARCH_ENABLED=true   # activates hybrid mode (BM25 + Qdrant)
HYBRID_SEARCH_ENABLED=false  # falls back to BM25-only (in-process, no Qdrant dependency)
```

When `HYBRID_SEARCH_ENABLED=false` or Qdrant is unreachable, the system falls back to BM25-only. No error is raised to callers; the degradation is logged at WARNING level.

## RRF Fusion Algorithm

Scores from BM25 and vector search are combined using **Reciprocal Rank Fusion (RRF)**:

```
rrf_score = (1 / (60 + rank_bm25)) + (1 / (60 + rank_vector))
```

Where:
- `rank_bm25` = 1-indexed position in the BM25 result list (1 = best BM25 match)
- `rank_vector` = 1-indexed position in the Qdrant vector result list
- Constant 60 is the standard RRF smoothing constant
- A document appearing in only one list gets a score contribution from that list only

Final results are sorted descending by `rrf_score`. Higher score = more relevant.

**What RRF does NOT use:** role, recency, workflow state, or any other signals. The ranking is purely positional fusion of BM25 and semantic similarity ranks.

---

## Fallback Degradation Path

```
HYBRID_SEARCH_ENABLED=true AND Qdrant reachable  → Full hybrid (BM25 + vector + RRF)
HYBRID_SEARCH_ENABLED=true AND Qdrant unreachable → BM25-only (logged at WARNING)
HYBRID_SEARCH_ENABLED=false                       → BM25-only (by configuration)
```

## Safety Rules

- `safe_for_customer=True` filter is enforced at both build time (BM25 index) and query time (Qdrant payload filter). Restricted content cannot reach LLM context via RAG.
- 18 unsafe chunks were purged from the index at the last build (as of Wave-2).
- `_RECENT_SEARCHES` ring buffer (deque, 200 items) is maintained in-process for incident debugging.

---

## What Was NOT Implemented

The article that previously existed here described a "5-signal ranking" (role match → keyword → semantic → recency → workflow state). **This was a design draft and was never implemented.** The production system uses RRF only. This article supersedes the previous description.
