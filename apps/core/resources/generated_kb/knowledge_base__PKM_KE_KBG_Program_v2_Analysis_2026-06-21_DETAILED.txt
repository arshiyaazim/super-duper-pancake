## REPORT 1 - EXECUTIVE SUMMARY

### 1.1 System Characterization
Fazle AI Platform is an identity-first operations intelligence stack that processes inbound messaging channels, performs role-aware routing, drives draft-first approvals, and orchestrates escort, attendance, payroll, payment, recruitment, and reporting workflows.

### 1.2 Core Conclusion
Production code currently contains non-trivial hidden business knowledge that is not fully represented in the current knowledge base. The knowledge base can only become the organizational brain after documenting hidden routing guards, parser logic, safety gates, state transitions, and decision thresholds.

### 1.3 Critical Outcomes
- Operational logic is highly deterministic and safety-gated.
- Role-based branches contain silent constraints not yet documented.
- Multiple hidden rules impact customer-facing behavior and risk posture.
- A conflict track is required before policy-level publication.

## REPORT 9 - WORKFLOW RECONSTRUCTION REPORT

### 9.1 Inbound Message Lifecycle
Actor: Contact or role holder
Input: Inbound bridge or meta message
Validation: dedup, cooldown, role detection, safety checks
Decision: role route, intent route, workflow route
Processing: draft or send
Database: message logs, draft queues, role lookups
Notification: admin/accountant as needed
Audit: command and incident logs

### 9.2 Escort Order Lifecycle
Actor: Escort client / admin
Flow: escort message parse -> draft creation -> admin completion/confirmation -> client delivery -> status updates

### 9.3 Release Closure Lifecycle
Actor: Employee/admin
Flow: release intent or OCR -> admin confirmation -> program close -> attendance backfill -> payment draft generation

### 9.4 Attendance Lifecycle
Actor: Supervisor/employee/admin
Flow: parse attendance -> create draft -> admin approve -> persist attendance

### 9.5 Recruitment Lifecycle
Actor: Candidate/unknown sender
Flow: eligibility check -> session create -> stepwise collection -> scoring -> stage transition

### 9.6 Admin Command Lifecycle
Actor: Admin role holder
Flow: parse -> dedup -> RBAC check -> execute -> audit
